Operating systems

Linux

I use two different Linux distributions. On my main PC, I enjoy gaming in my free time, so I use Linux Mint 20.1. It suits my use case while avoiding the hell that is Windows and things I don't like about Canonical (and Ubuntu by extension), namely their attempts to 'corporatise' Linux, their push for Snaps, and their "Ubuntu Software Store" which is scarily similar to the walled gardens of the Mac OS app store.

On my laptop, which is primarily for work, I use Artix Linux. Artix is essentially Arch without systemd. I don't hate systemd, I just prefer runit, though I have not tried OpenRC or s6 which are two other init systems offered in a base install of Artix.

Really, you can accomplish anything with any Linux distribution, with enough time and patience. I would tend to recommend Arch-based distros above all others, though I suppose personal preference trumps all.

Windows

I have a Windows dual-boot on my laptop and PC for the occasions where a piece of software demands it, or I need to use .NET Framework for something. Other than that, I don't touch it.

Utilities

Suckless tools

I use DWM (dynamic window manager) as my window manager of choice on my laptop. DWM is infinitely customisable by its very nature. You are intended to overwrite, modify, and extend the source code to fit your individual needs, or install extra features others have already made for you. All Suckless offer is the base code and an empty palette for you to work on.

My terminal emulator of choice is ST (simple terminal). Like DWM, it is made by Suckless and is also offered as a modifiable base upon which to build your own release.

Web browser

Brave, I believe, is the best web browser out there at the moment. It's fast, contains almost every privacy feature after changing a couple of settings, supports web extensions for Chrome, and is fully open-source. Chromium is okay, though I would recommend Ungoogled Chromium over the normal version, but Brave is the simplest solution.

I used to use Mozilla Firefox, but it relies on Google's money, and Mozilla has increasingly abandoned their routes as of late. It's still a decent option for a browser, and the biggest non-Chromium based browser left, but it's falling behind in terms of speed.

Text editors

For all-purpose text editing and automation I use Vim. Vim is great. It's lightweight, powerful, terminal based, and is very efficient when you learn to use it.

For coding projects that don't require a full IDE, VSCodium is my go-to. If Degoogled Chromium is Chromium without Google, VSCodium Demicrosofted VS Code. VS Code is open-source, but the binary download available on MS' website is closed-source, and contains Microsoft's telemetry (data collection). VSCodium takes the great features of VS Code and strips the spookiness out of it.

Lastly, my IDE of choice is JetBrains Rider. It, unfortunately, isn't open-source. It also isn't free, but it is available on Linux, and I get it for free as a student. If a fully-featured, free and open-source .NET IDE becomes available, I will likely switch to it.

Multimedia

The single best image viewer on Linux is SXIV (Simple X Image Viewer). It's feature-rich, extremely lightweight, very fast, and fully open-source.

There are lots of options for video players, but the one I use is mpv. Terminal-based, fast, open-source. I've always had technical issues with VLC, and I have no need for the advanced features, so I don't use it.
